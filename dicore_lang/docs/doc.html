Dicore Programming Language 🚀

Welcome to Dicore, a custom programming language designed to explore compiler development, language design, and software architecture. It uses LLVM to generate machine code and currently supports basic token parsing and expression evaluation.

🛠 Project Structure

Folders

  • frontend/: Contains code for parsing and interpreting the source code.
  • backend/: Contains code for interacting with LLVM to generate machine code.

📂 Frontend Components

  • Lexer (lexer.h & lexer.cpp): Converts source code into tokens (numbers, operators, keywords).
  • Token (token.h & token.cpp): Categorizes the tokens (e.g., numbers, operators).
  • AST (Abstract Syntax Tree) (ast.h & ast.cpp): Represents the structure of parsed code.
  • Parser (parser.h & parser.cpp): Converts tokens into an AST while ensuring syntactical correctness.

🔧 Backend Components

  • Compiler (compiler.h & compiler.cpp): Uses LLVM to convert the AST into LLVM Intermediate Representation (IR) and generates machine code.
  • CMakeLists.txt: Configures the build system with CMake and links necessary dependencies.

🚀 Getting Started

1. Install Dependencies

You'll need LLVM and CMake to compile and run the Dicore language.

  • Install LLVM on macOS using Homebrew: 
    brew install llvm@14
  • Ensure LLVM tools are in your PATH by adding: 
    export LDFLAGS="-L/opt/homebrew/opt/llvm@14/lib"
    export CPPFLAGS="-I/opt/homebrew/opt/llvm@14/include"

2. Build the Project

After setting up dependencies, build the project using CMake:

mkdir build
cd build
cmake ..
make

This will generate the executable.

3. Running the Compiler

To run the Dicore compiler with your source code, use the following command:

./dicore_lang path_to_source_code.dicore

The compiler will generate machine code from your .dicore file.

✅ What I've Accomplished So Far

✔️ Completed Features

  • Lexer: Tokenizes source code into meaningful components.
  • Parser: Converts tokens into an Abstract Syntax Tree (AST).
  • AST: Represents code structure.
  • Compiler: Uses LLVM to generate machine code from the AST.

🛑 Current Limitations

  • Currently, the language only supports basic expressions (numbers).
  • It doesn't support advanced features like functions, conditionals, or loops yet.
